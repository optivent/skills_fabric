# OpenCode Plugin Template Generator
[](https://github.com/zenobi-us/opencode-plugin-template#opencode-plugin-template-generator)
ðŸŽ‰ Generate a new OpenCode plugin in seconds!
This is a generator repository for creating new [OpenCode](https://opencode.ai) plugins. It provides a starter template with all the scaffolding you need to build powerful OpenCode plugins.
## Quick Start
[](https://github.com/zenobi-us/opencode-plugin-template#quick-start)
### 1. Use this template
[](https://github.com/zenobi-us/opencode-plugin-template#1-use-this-template)
Click "Use this template" on GitHub or clone it:
```
git clone https://github.com/zenobi-us/opencode-plugin-template.git my-plugin
cd my-plugin
```

### 2. Run the generator
[](https://github.com/zenobi-us/opencode-plugin-template#2-run-the-generator)
```
./setup.sh
```

### 3. Answer the prompts
[](https://github.com/zenobi-us/opencode-plugin-template#3-answer-the-prompts)
The generator will ask for:
  * **Plugin name** - kebab-case identifier (e.g., `my-awesome-plugin`)
  * **Description** - What your plugin does
  * **Author name** - Your name
  * **Author email** - Your email
  * **Repository URL** - GitHub repo URL
  * **GitHub org/username** - For workflow configuration


### 4. Start developing!
[](https://github.com/zenobi-us/opencode-plugin-template#4-start-developing)
```
cd my-plugin
bun install
mise run build
```

## What You Get
[](https://github.com/zenobi-us/opencode-plugin-template#what-you-get)

## Usage
[](https://github.com/zenobi-us/opencode-plugin-template#usage)
```
# Generate a new plugin (interactive prompts)
./setup.sh generate

# Show help
./setup.sh help

# Show version
./setup.sh version
```

## Development
[](https://github.com/zenobi-us/opencode-plugin-template#development)
### Available Scripts
[](https://github.com/zenobi-us/opencode-plugin-template#available-scripts)
```
bun install          # Install dependencies
mise run setup       # Initial setup
mise run build       # Build the plugin
mise run test        # Run tests
mise run lint        # Lint code
mise run lint:fix    # Fix linting issues
mise run format      # Format code with Prettier
mise run pkgjsonlint # Lint package.json
mise run prepare     # Prepare for release
mise run publish     # Publish the plugin
mise run version     # Manage version
```

### Publishing
[](https://github.com/zenobi-us/opencode-plugin-template#publishing)
See [RELEASE.md](https://github.com/zenobi-us/opencode-plugin-template/blob/main/template/RELEASE.md) for publishing and release management details.
**TL;DR:** Push single commits to main with [conventional commit format](https://www.conventionalcommits.org/). Release-please will accumulate changes in a release PR. When this release PR is merged, a new minor version is released. Until then, all other commits on main result in patch builds being published.
