### AST-Aware Code Analysis and Transformation (AST-grep)
link

The system integrates AST-grep tools for structural code analysis and transformation, offering functionalities for pattern searching, code rewriting, environment verification, and binary management. It also supports direct Abstract Syntax Tree (AST) manipulation through [`@ast-grep/napi`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/bun.lock#L9).
The core AST-grep functionalities are exposed as [`ast_grep_search`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L70) and [`ast_grep_replace`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L71) tools. [`ast_grep_search`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L70) allows users to locate code patterns using meta-variables (e.g., [`$VAR`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/AGENTS.md?plain=1#L59) for single nodes, [`$$$`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/AGENTS.md?plain=1#L59) for multiple nodes), facilitating flexible matching across various programming languages. It also provides hints for common pattern-related errors. [`ast_grep_replace`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L71) enables AST-aware code rewriting by matching a [`pattern`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/hooks/ralph-loop/index.ts#L84) and applying a [`rewrite`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/ast-grep/cli.ts#L19) pattern, with support for a [`dryRun`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/ast-grep/utils.ts#L66) mode to preview changes before execution.
The integration includes robust management of the [`ast-grep`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/bun.lock#L8) CLI binary. This involves ensuring the binary's availability, handling its download from GitHub releases, caching it locally, and extracting it for execution across different operating systems. The system determines the appropriate platform-specific binary and manages executable permissions.
For more direct AST manipulation, the system integrates with [`@ast-grep/napi`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/bun.lock#L9) via [`src/tools/ast-grep/napi.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/ast-grep/napi.ts). This allows for parsing code into an AST, finding patterns, extracting metavariables, and applying rewrites programmatically. The system maps supported languages (e.g., Python, JavaScript, TypeScript, TSX) to [`Lang`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/ast-grep/napi.ts#L1) objects for parsing and transformation.
Environment checks are performed to ensure the [`ast-grep`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/bun.lock#L8) CLI is correctly installed and configured, providing diagnostics and status reporting. The system includes mechanisms to execute [`ast-grep`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/bun.lock#L8) commands with timeouts and handles partial or truncated JSON output, attempting to extract usable information even when output limits are reached. This includes logic to re-attempt execution after binary download if an [`ENOENT`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/cli/config-manager.ts#L31) error occurs, providing a self-healing capability.
For more details on the command-line interface, refer to [Command-Line Interface (CLI) Management](https://codewiki.google/github.com/code-yeongyu/oh-my-opencode#command-line-interface-cli-management). The overall structure of built-in tools is described in [Built-in Tools and Utilities](https://codewiki.google/github.com/code-yeongyu/oh-my-opencode#built-in-tools-and-utilities).
### Background Task Orchestration
link

The system includes tools for managing asynchronous background tasks, encompassing their initiation, output retrieval, cancellation, lifecycle management, and status formatting. These tools allow AI agents to offload long-running operations and interact with them in a structured manner.
Tasks are initiated using the [`createBackgroundTask`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/tools.ts#L48) tool, which takes a description, a detailed prompt, and the agent type to be used. This tool provides immediate feedback to the calling agent, including a unique task ID and session ID, along with instructions on how to monitor its progress or retrieve its output. This mechanism enables multi-phase operations where an orchestrator agent can delegate specialized work to other agents, facilitating [Agent Orchestration and Core System](https://codewiki.google/github.com/code-yeongyu/oh-my-opencode#agent-orchestration-and-core-system).
To check the status or retrieve the final output of a previously launched background task, agents use the [`createBackgroundOutput`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/tools.ts#L235) tool. This tool supports both non-blocking retrieval of the current status and blocking retrieval that waits for task completion, with a configurable timeout. Upon completion, the tool fetches and formats the final output, typically the last assistant message, for the calling agent.
The [`createBackgroundCancel`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L39) tool allows for the termination of running background tasks. It can target a specific task by its ID or cancel all descendant tasks associated with the current session. This provides agents with control over their delegated work, enabling them to stop tasks that are no longer necessary or have gone awry.
The framework also includes utility functions to assist in managing these tasks. For instance, [`formatTaskStatus`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/tools.ts#L113) generates a markdown-formatted string displaying the task's current status and details, including duration, last tool used, and the original prompt. [`formatDuration`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/tools.ts#L25) converts time durations into human-readable strings, while [`truncateText`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/tools.ts#L108) handles the shortening of lengthy text outputs for conciseness. These utilities, primarily found in [`src/tools/background-task/tools.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/tools.ts), ensure that agents receive clear and concise information about their background operations. The types used for defining background task arguments, such as [`BackgroundTaskArgs`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/types.ts#L1), [`BackgroundOutputArgs`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/types.ts#L7), and [`BackgroundCancelArgs`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/types.ts#L13), are centrally defined in [`src/tools/background-task/types.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/types.ts) to ensure type safety and consistency across the system. The constants used for tool descriptions are defined in [`src/tools/background-task/constants.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/constants.ts). The main entry point for these tools is [`src/tools/background-task/index.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/background-task/index.ts).
### Inter-Agent Communication: call_omo_agent
link

The [`call_omo_agent`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L538) tool provides a mechanism for AI agents to delegate tasks to other specialized subagents, facilitating multi-agent collaboration within the system. This tool supports both synchronous and asynchronous execution modes for these delegated tasks, along with session management and argument validation.
Agents can use [`call_omo_agent`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L538) to initiate tasks with a specified [`subagent_type`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/types.ts#L8) and a [`prompt`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/index.ts#L269). The core functionality is implemented within the [`createCallOmoAgent`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/tools.ts#L15) function in [`src/tools/call-omo-agent/tools.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/tools.ts), which generates a [`ToolDefinition`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/lsp/tools.ts#L1) that agents can invoke.
The [`call_omo_agent`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L538) tool offers two primary execution modes for subagent interaction:
  * **Asynchronous Background Execution** : When [`run_in_background`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/types.ts#L9) is set to [`true`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/plugin-handlers/config-handler.ts#L111), the subagent task is launched asynchronously. The calling agent receives an immediate [`Task ID`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/agents/sisyphus.ts#L285) and [`Session ID`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/cli/run/types.ts#L24), allowing it to continue processing without waiting for the subagent's completion. This mode leverages a [`BackgroundManager`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/features/background-agent/manager.ts#L57) to orchestrate the task.
  * **Synchronous Execution** : If [`run_in_background`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/types.ts#L9) is [`false`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/cli/index.ts#L100), the calling agent pauses its own execution until the subagent task is completed. This mode involves the creation of a new session or the continuation of an existing one if a [`session_id`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/hooks/ralph-loop/storage.ts#L50) is provided. The [`OpenCode`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L123) client then sends the [`prompt`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/index.ts#L269) to the subagent within this session and retrieves its response. During synchronous subagent prompting, certain tools like [`task`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/agents/oracle.ts#L106), [`call_omo_agent`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L538), and [`background_task`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/agents/oracle.ts#L106) are explicitly disabled to prevent recursive or unintended tool usage by the subagent. The results of synchronous calls include the [`session_id`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/hooks/ralph-loop/storage.ts#L50) within [`<task_metadata>`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/tools.ts#L194) tags for structured parsing by the calling system.

Argument validation is performed using a dynamically generated schema derived from [`tool.schema`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/lsp/tools.ts#L40) to ensure that [`subagent_type`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/types.ts#L8), [`prompt`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/index.ts#L269), [`run_in_background`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/types.ts#L9), and [`session_id`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/hooks/ralph-loop/storage.ts#L50) are correctly provided. Type definitions for [`AllowedAgentType`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/types.ts#L3), [`CallOmoAgentArgs`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/types.ts#L5), and [`CallOmoAgentSyncResult`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/types.ts#L13) are centralized in [`src/tools/call-omo-agent/types.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/types.ts), ensuring type safety for agent selection and result handling. Constants, such as [`ALLOWED_AGENTS`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/constants.ts#L1) and [`CALL_OMO_AGENT_DESCRIPTION`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/constants.ts#L3), are defined in [`src/tools/call-omo-agent/constants.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/call-omo-agent/constants.ts) to manage valid subagent types and tool descriptions. ### File System Search and Globbing Tools
link

The system provides two primary tools for navigating and understanding the codebase: [`grep`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L72) for content searching and [`glob`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L73) for file path pattern matching. Both tools prioritize efficiency and resource management, leveraging external command-line utilities like [`ripgrep`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/grep/downloader.ts#L142) for performance.
The [`grep`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L72) tool performs fast, regex-based content searches within files, supporting parameters for specifying the regex pattern, including or excluding file patterns, and defining the search path. To prevent excessive resource consumption, searches are time-limited (60 seconds) and output is capped at 10MB. The system can automatically detect, install, and manage the [`ripgrep`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/grep/downloader.ts#L142) binary, preferring it over standard [`grep`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L72) due to its performance benefits, as detailed in files like [`src/tools/grep/cli.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/grep/cli.ts) and [`src/tools/grep/downloader.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/grep/downloader.ts). Search results are formatted for readability, providing details like file paths, line numbers, and matched text, with options for sorting by modification time as seen in [`src/tools/grep/utils.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/grep/utils.ts).
The [`glob`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L73) tool, defined in [`src/tools/glob/tools.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/glob/tools.ts), enables users to find files based on specific glob patterns (e.g., [`**/*.js`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L571)) within a designated directory. The results are formatted to present matching file paths, optionally sorted by modification time, and indicate if the results were truncated. The system defines standard structures for [`FileMatch`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/glob/types.ts#L1), [`GlobResult`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/glob/types.ts#L6), and [`GlobOptions`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/glob/types.ts#L13) in [`src/tools/glob/types.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/glob/types.ts) to ensure consistent handling of file matching operations. Both [`grep`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L72) and [`glob`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/index.ts#L73) rely on the underlying [`ripgrep`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/grep/downloader.ts#L142) executable, ensuring a consistent and optimized approach to file system interactions.
### Interactive Shell Access with Tmux
link

The [`interactive_bash`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/interactive-bash/tools.ts#L50) tool provides a mechanism for agents to execute Bash commands within dedicated [`tmux`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L186) sessions. This is particularly useful for tasks requiring persistent processes or interactive command-line tools, as it allows the agent to interact with a consistent environment across multiple steps. The system ensures secure execution, manages command parsing, handles potential timeouts, and efficiently locates and manages the [`tmux`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L186) executable.
A key aspect of this tool is its secure handling of commands. It processes the input [`tmux`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L186) command string, tokenizing it to correctly interpret arguments, including those with quotes or escape sequences. Before execution, the system performs a security check, blocking specific [`tmux`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L186) subcommands (e.g., those related to buffer capture or piping) to prevent misuse or unintended side effects, as defined in [`src/tools/interactive-bash/constants.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/interactive-bash/constants.ts). This encourages the use of other specialized tools for tasks like reading terminal output, maintaining a clear separation of concerns within the agent's toolkit.
The execution of commands is managed within [`tmux`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L186) sessions, which are identified by an "omo-{name}" pattern. This approach allows for the execution of long-running processes or interactive CLI tools without blocking the agent's main thread. To prevent indefinite execution, each command includes a timeout mechanism, which terminates processes that exceed a predefined duration.
Efficient management of the [`tmux`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L186) executable path is also integral to the tool's operation. The system includes utilities that asynchronously detect and verify the [`tmux`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L186) binary on the system. This path is then cached to avoid redundant searches, ensuring that subsequent calls to the [`interactive_bash`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/interactive-bash/tools.ts#L50) tool can quickly access the [`tmux`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/README.md?plain=1#L186) executable. This lazy initialization and memoization pattern, implemented in [`src/tools/interactive-bash/utils.ts`](https://github.com/code-yeongyu/oh-my-opencode/blob/d0694e5aa4ecd5e0179718e190136f8b213200d9/src/tools/interactive-bash/utils.ts), optimizes performance by minimizing system calls.
